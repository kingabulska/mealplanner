Description

Planning the meals is only half of the job: we also need to make sure that all the ingredients are bought ahead of time! In this stage, the program should generate a shopping list containing all the required ingredients for the planned meals. If some ingredient is needed for several meals, put it in the list only once in the following format: ingredient xN, where N is the number of times this ingredient occurs.

Finally, let's add the option of saving the list to the file. Add the command save to the menu. Save the shopping list to the file, unless the plan hasn't been finished: then, print an appropriate message. In the end, don't forget to ask the user for the file name.

Objectives

1. Ask the user about filename with the message Write a filename:.

2. Save option is only allowed after the meals have been planned for the week. If the user didn't plan meals, print Save isn't allowed. Plan your meals first..

3. The format of the shopping list should be as follows:

eggs
tomato x3
beef
broccoli
salmon
chicken x2

5. When the list has been saved, print the message Saved!.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1

Note: the example below assumes that the database has quite a few meals stored and that the plan command has already been executed.

What would you like to do (add, show, plan, save, exit)?
> save
Write a filename:
> shoppinglist.txt
Saved!
What would you like to do (add, show, plan, save, exit)?
> exit
Bye!

Example 2

Note: the example below assumes that the database has quite a few meals stored but the plan command hasn't been executed yet.

What would you like to do (add, show, plan, save, exit)?
> save
Save isn't allowed. Plan your meals first.
What would you like to do (add, show, plan, save, exit)?
> exit
