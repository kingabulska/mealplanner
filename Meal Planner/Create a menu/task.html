Description

One meal is not going to get you far. Let's create a menu that allows the user to add several meals and show them together with their properties. For this, we need to add a few commands.

  • The command add should start the meal input process and prompt the user for the meal's properties.
  • The command show should print the list of all the saved meals with their names, categories, and ingredients.
  • After executing add or show, the program should ask the user about the next action.
  • The program should run until the user inputs exit, which terminates the program.

In this stage, your program should also check the user input: what if the user entered something wrong?

  • If the input meal category is something other than breakfast, lunch, or dinner, print Wrong meal category! Choose from: breakfast, lunch, dinner.
  • If the meal's name or ingredients have the wrong format (for example, if they contain numbers), print Wrong format. Use letters only!.

Objectives

  1. Make an infinite loop that can only be terminated with the exit command.

  2. The program should prompt the user to choose the action with the message What would you like to do (add, show, exit)?

  3. After the command has been processed, the user should be prompted for another action.

  4. Make sure that the input and output format is correct.

  5. If the user asks the program to show meals before any were added, print No meals saved. Add a meal first.

  6. After the exit command, print Bye! and end the program.

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
